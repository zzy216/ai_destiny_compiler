## 三、功能分析

### 范围和受影响组件
- 种子数据定义 — 提供固定 UUID、模型、教练和知识卡内容。
- 种子执行器 — 校验环境、运行 Migration、在事务中幂等插入基础数据。
- API package scripts — 提供 `pnpm seed` 命令。
- 测试和文档 — 验证安全约束、幂等定义和执行结果，并同步执行清单。

### 行为和数据流
读取环境变量 → 拒绝生产环境或无效密码 → 初始化 DataSource → 执行未完成 Migration → 开启事务 → 按固定标识创建用户、配置和知识卡 → 提交事务 → 关闭 DataSource。

### 边界条件和失败处理
- 缺少或过短种子密码 — 在连接数据库前失败。
- 生产环境 — 在连接数据库前拒绝执行。
- 已存在账号 — 保留已有密码和账号状态，只补齐缺失的 Profile。
- 已存在配置或知识卡 — 按固定标识复用，不重复插入。
- 事务中任一记录失败 — 回滚本批次写入。

### 风险和兼容性
- 密码处理 — 使用 Argon2id；不在日志、输出或代码中打印密码。
- 模型凭据 — 只创建无真实 Key 的占位配置，不创建明文凭据记录。
- 外键顺序 — 先管理员和模型，再创建依赖模型的教练配置；知识卡引用管理员。
- 重复执行 — 使用固定 UUID、slug 和 `(card_key, version)` 幂等查找。

### 方案决策
使用独立的 TypeScript seed CLI，而不是把种子数据混入 Migration。这样开发环境可以显式执行、生产环境可以拒绝，并能从环境变量读取初始密码；数据库结构 Migration 仍保持只负责结构变更。
