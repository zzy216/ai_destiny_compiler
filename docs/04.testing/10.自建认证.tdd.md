## 用户旅程

1. 作为普通用户，我希望用邮箱或用户名登录，以便后续会话、模型和消息都绑定到我的账号。
2. 作为受邀用户，我希望用一次性邀请码注册，以便系统只开放受控账号入口。
3. 作为已登录用户，我希望 Refresh Token 被轮换并支持登出和改密失效，以便旧 Token 不能长期复用。
4. 作为管理员，我希望只有 admin 角色能访问管理后台和后台 API，以便普通用户不能维护系统模型、教练配置或查看运营摘要。
5. 作为管理员，我希望 React 后台先登录再加载数据，以便所有后台请求携带真实认证上下文。

## 已知缺口

- React 管理后台本阶段将 Token 保存在内存中，刷新页面后需重新登录；未把管理员会话升级到 HttpOnly Cookie + CSRF。
- 限流为单进程固定窗口，不适合多实例共享；后续生产部署可替换为 Redis 或网关限流。
- 未实现密码找回、验证码、邮箱验证、管理员重置密码和审计日志。
