- [数据库设计](../02.technology/02.数据库设计.md)

## User journeys

1. 作为后端开发者，我希望应用使用 PostgreSQL 且关闭 `synchronize`，以便生产结构只通过 Migration 演进。
2. 作为后端开发者，我希望数据库设计中的核心实体全部注册到 TypeORM，以便后续 Service 可以使用 Repository。
3. 作为后端开发者，我希望 20 个 Migration 按设计顺序可执行，以便空数据库能够完整创建认证、模型、会话和行动闭环表。

## TDD execution

### RED

新增 `apps/api/test/database.contract.e2e-spec.ts` 后首次执行：

```text
pnpm test -- database.contract.e2e-spec.ts
FAIL: Cannot find module 'typeorm' / database.config / database/entities / database/migrations
```

失败原因是数据库接入尚未实现，符合预期的 RED 状态。

### GREEN

实现 TypeORM 配置、17 个 Entity、20 个 Migration 和 NestJS DatabaseModule 后执行：

```text
pnpm test -- database.contract.e2e-spec.ts
PASS: 2 tests
```

在本机临时 PostgreSQL 数据库中执行：

```text
DB_HOST=127.0.0.1 DB_PORT=5432 DB_USERNAME=zzy \
DB_NAME=destiny_compiler_migration_20260716 pnpm migration:run
```

实际校验结果：

- 17 张业务表创建成功；
- 20 条 Migration 记录写入 `typeorm_migrations`；
- 45 条外键约束创建成功；
- 临时数据库已删除；
- 当前本地 `destiny_compiler` 数据库保留已执行的 17 张表和 20 条 Migration 记录。
